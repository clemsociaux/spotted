FAQ

Questions you'll ask
before we answer

Still not sure? Chat with us anytime.

Is WebhookInbox free to use?
Yes. The Starter plan is free forever with 1 endpoint and 7 days of history. No credit card required to sign up. Upgrade only when you need more.
Does it work with any webhook provider?
Absolutely. Any service that sends HTTP POST requests works — Stripe, Shopify, GitHub, Auth0, Twilio, Paddle, or your own system. If it sends webhooks, WebhookInbox receives them.
How does the replay feature work?
Replay sends a real HTTP POST from our server to any URL you specify, using the original payload and headers. You see the full response — status code, response body, and timing. It's the same as re-triggering the original event.
Can I use it during local development?
Yes. Point your webhook provider to your WebhookInbox URL, inspect the payloads, then use Replay to forward them to your localhost. No tunnel required for inspection.
Is my data secure and private?
Each endpoint URL is cryptographically unique and unguessable. Only your account can view received events. Data is encrypted in transit and at rest. Events are purged after your retention period.
What are alert rules exactly?
Alert rules let you define conditions on webhook fields — for example "notify me if data.amount is greater than 1000" or "alert on any payment.failed event". Notifications appear in-app immediately.
What stack is it built on?
WebhookInbox is built with PHP 8 + Slim Framework for the API, MySQL for storage, and vanilla JS for the frontend. No heavy dependencies. Simple, fast, and reliable.
Can I export or download my events?
You can copy any payload as JSON or as a cURL command directly from the interface. Full CSV/JSON export is available on Pro and Team plans. API access for programmatic export coming soon.
